## 📌 Project Overview

A full-stack web application that empowers Indian farmers with AI-driven tools:
- 🌾 **Crop Recommendation** using soil & climate data (Random Forest ML)
- 🔬 **Disease Detection** from leaf photos (CNN / MobileNetV2)
- 🏡 **Land Price Prediction** (Gradient Boosting regression)
- ⛅ **Live Weather** with farming advisories (OpenWeatherMap API)
- 📊 **Market Price Tracker** (mandi rates for 50+ crops)
- 🧪 **Soil Analysis** with fertilizer recommendations
- 📋 **Government Schemes** information
- 🌐 **Telugu Language** support

## 🚀 Quick Start

### Step 1: Install Python dependencies
```bash
cd backend
pip install -r requirements.txt
```

### Step 2: Train ML models
```bash
python ml_models/train_models.py
# This creates crop_model.pkl and land_model.pkl
```

### Step 3: Set environment variables
Create a `.env` file in the `backend/` folder:
```
OPENWEATHER_API_KEY=your_free_api_key_here
```
Get free key at: https://openweathermap.org/api

### Step 4: Start the backend server
```bash
cd backend
uvicorn main:app --reload --port 8000
```
API is now live at: http://localhost:8000  
Auto-docs at: http://localhost:8000/docs ← (Swagger UI — test all endpoints)

### Step 5: Open the website
```
Open frontend/index.html in any browser
```
The website works standalone as a demo.  
For live AI predictions, it calls the backend on port 8000.

## 🤖 ML Models Explained

### 1. Crop Recommendation (Random Forest)
- **Algorithm:** Random Forest Classifier (200 trees)
- **Input:** 7 features (N, P, K, pH, temperature, humidity, rainfall)
- **Output:** Crop label + confidence % + fertilizer tip
- **Dataset:** Kaggle Crop Recommendation Dataset (2,200 rows, 22 crops)
- **Accuracy:** ~99.3%

### 2. Disease Detection (CNN)
- **Architecture:** MobileNetV2 (Transfer Learning from ImageNet)
- **Input:** 224×224 RGB leaf image
- **Output:** Disease class + confidence + treatment advice
- **Dataset:** PlantVillage (54,309 images, 38 classes)
- **Accuracy:** ~96.5% validation accuracy
- **Training:** Run on Google Colab (free GPU) — ~1.5 hours

### 3. Land Price (Gradient Boosting)
- **Algorithm:** GradientBoostingRegressor (200 estimators)
- **Input:** State, area, soil type, irrigation, road distance
- **Output:** Total value (INR) + per-acre rate
- **R² Score:** ~0.91

## 🌐 Connecting Frontend to Backend

In `frontend/index.html`, the prediction functions call the backend via `fetch()`.
Once backend is running on port 8000, update the frontend to use real API:

```javascript
// In predictCrop() function — replace the mock with:
const response = await fetch('http://localhost:8000/api/predict/crop', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    nitrogen:    parseFloat(v('crop-n')),
    phosphorus:  parseFloat(v('crop-p')),
    potassium:   parseFloat(v('crop-k')),
    ph:          parseFloat(v('crop-ph')),
    temperature: parseFloat(v('crop-temp')),
    humidity:    parseFloat(v('crop-hum')),
    rainfall:    parseFloat(v('crop-rain'))
  })
});
const data = await response.json();
id('crop-result-val').textContent = data.crop;
id('crop-result-sub').textContent = data.fertilizer_tip;
```

## ☁️ Deployment Guide

### Option A: Free Deployment (Render.com)
1. Push code to GitHub
2. Go to render.com → New Web Service
3. Connect GitHub repo, select `backend/` folder
4. Build command: `pip install -r requirements.txt`
5. Start command: `uvicorn main:app --host 0.0.0.0 --port $PORT`
6. Add environment variable: `OPENWEATHER_API_KEY`

### Option B: VPS (DigitalOcean / AWS)
```bash
# On server:
git clone your-repo
cd smart-agri/backend
pip install -r requirements.txt
python ml_models/train_models.py
uvicorn main:app --host 0.0.0.0 --port 8000 &
```

### Frontend: Deploy to GitHub Pages
1. Push `frontend/index.html` to GitHub
2. Settings → Pages → Deploy from main branch
